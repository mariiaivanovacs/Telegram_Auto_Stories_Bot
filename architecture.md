# Архитектура: Telegram Price Monitor + Story Bot

## Обзор системы

Бот состоит из двух основных частей:

1. **Userbot (Telethon)** — читает сообщения из конкурентских Telegram-каналов от имени обычного пользователя.
2. **Admin-бот (python-telegram-bot)** — принимает команды от владельца, запускает пайплайн, рассылает результаты и изображения для сторис.

Оба компонента работают в одном Docker-контейнере и общаются через общую SQLite-базу и объекты Python в памяти.

---

## Схема пайплайна

```
[APScheduler / ручной запуск]
         │
         ▼
   fetcher.py          ← Telethon читает сообщения за сегодня из каналов
         │
         ▼
   parser.py           ← нормализация текста, транслитерация рус→лат
         │
         ▼
   matcher.py          ← regex-матчинг продукта + извлечение цены
         │
         ▼
   pricing.py          ← min(конкурент) − 500 ₽, флаг больших изменений
         │
     [≥3000 ₽?]────────► admin подтверждает / отклоняет через inline-кнопки
         │
         ▼
   story.py            ← Pillow: фон + blur + тёмный оверлей + текст
         │
         ▼
   sender.py           ← отправляет цены + отчёт + изображения в Telegram
         │
         ▼
   db.py               ← сохраняет историю цен в SQLite
```

## Запуск (пошаговая инструкция)

### Требования

- Python 3.11+ (локально, только для создания сессии)
- Docker + docker-compose (на сервере)
- Аккаунт на https://my.telegram.org (API ID и API Hash)
- Бот-токен от @BotFather
- Ваш Telegram user ID (можно узнать у @userinfobot)

---

### Шаг 1 — Скопировать конфиги

```bash
cp .env.example .env
cp config.example.yaml config.yaml
```

Заполнить `.env`:

| Переменная           | Что вписать                                      |
|----------------------|--------------------------------------------------|
| `TELEGRAM_API_ID`    | API ID с https://my.telegram.org                 |
| `TELEGRAM_API_HASH`  | API Hash с https://my.telegram.org               |
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather                         |
| `TELEGRAM_ADMIN_ID`  | Ваш числовой Telegram user ID                    |
| `TELEGRAM_PHONE`     | Номер телефона аккаунта для userbot (+79...)      |
| `ADMIN_PASSWORD`     | Любой пароль для команды /admin                  |

Заполнить `config.yaml`:
- `channels` — список конкурентских каналов (`@username`)
- `admins` — ваш `telegram_id` и `username`
- Остальные параметры (скидка, порог изменений, расписание) можно оставить как есть

---

### Шаг 2 — Создать userbot-сессию (один раз)

```bash
pip install telethon python-dotenv pyyaml
python scripts/create_session.py
```

Скрипт запросит код из Telegram и сохранит файл `data/userbot.session`.  
Этот файл **не коммитится** в git — храните его в безопасном месте.

---

### Шаг 3 — Положить фоны для сторис

Скопируйте `.JPG`-файлы в папку `backgrounds/`.  
Без фонов сторис не генерируются — бот предупредит об этом при запуске пайплайна.

---

### Шаг 4 — Запустить контейнер

```bash
docker compose up -d --build
```

Проверить логи:

```bash
docker compose logs -f
```

---

### Обновление

```bash
git pull
docker compose up -d --build
```

## Принятые архитектурные решения

### Почему нет интеграции с AI

Парсинг цен реализован через regex + транслитерацию без использования языковых моделей.  
**Причина:** LLM-интеграция добавляет внешнюю зависимость, задержки, расходы и непредсказуемость ответов. Для структурированных числовых данных (цена + модель) regex надёжнее, быстрее и полностью предсказуем. AI можно добавить позже как запасной слой для сложных случаев.

### Почему нет автоматического сбора данных

Бот читает каналы только по команде администратора или по расписанию раз в сутки в 09:00 МСК.  
**Причина:** постоянный мониторинг в реальном времени увеличивает нагрузку на Telegram API и риск блокировки аккаунта. Ежедневная сводка достаточна для ценообразования.

### Почему нет автопостинга в сторис

Сгенерированные изображения отправляются владельцу в Telegram — но **не постятся автоматически**.  
**Причина:** автоматическая публикация в Telegram Stories через неофициальный MTProto API нестабильна и может привести к бану аккаунта. Владелец проверяет изображения и публикует их вручную. Это осознанное решение в пользу безопасности аккаунта.

## Ограничения текущей версии

1. **Хрупкое извлечение цен.** Regex-матчер работает хорошо для стандартных форматов, но легко ломается при нестандартном оформлении сообщений у конкурентов — таблицах, картинках с текстом, нестандартных сокращениях.

2. **Пропуск изменений цен.** Если конкурент обновил прайс внутри дня (после 09:00), бот узнает об этом только следующим утром. Мониторинг не в реальном времени — это осознанный компромисс.

3. **Одна Telegram-сессия.** Весь доступ к каналам идёт через один аккаунт. При смене номера или блокировке нужно заново создавать `userbot.session`.

4. **SQLite без репликации.** База данных хранится в одном файле внутри volume. Нет автоматического бэкапа — при потере volume теряется история цен.

5. **Только текстовые сообщения.** Бот не обрабатывает прайс-листы в виде изображений, PDF или пересланных файлов. Если конкурент публикует цены в фото — они не попадут в выборку.

6. **Жёстко зашитый список продуктов.** Добавить новый продукт можно только через редактирование `config.yaml` и перезапуск контейнера — нет UI для добавления продуктов.

## Возможные улучшения (будущие версии)

| Улучшение | Описание |
|---|---|
| AI-слой для парсинга | GPT/Claude как fallback для сообщений, которые не распарсил regex |
| OCR для прайс-картинок | Tesseract или cloud Vision API для извлечения цен из фотографий |
| Авто-постинг сторис | После стабилизации MTProto Stories API — добавить опциональный автопостинг с явным подтверждением |
| Мониторинг в реальном времени | Подписка на новые сообщения через Telethon event handler (с rate-limit защитой) |
| PostgreSQL вместо SQLite | Для масштабирования и встроенного бэкапа |
| UI для управления продуктами | Добавление / удаление / редактирование продуктов прямо из Telegram-бота |
| Алерты на изменения у конкурентов | Push-уведомление при обнаружении сброса цены на ≥ X% в течение дня |
| Веб-дашборд | Простая страница с историей цен и графиками (FastAPI + Chart.js) |
